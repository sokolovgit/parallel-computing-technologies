# ВИСНОВКИ

У ході виконання курсової роботи було досліджено алгоритм бітонічного сортування та особливості його паралельної реалізації на багатоядерних обчислювальних системах. У теоретичній частині розглянуто принцип побудови бітонічних послідовностей, структуру сортувальної мережі та причини, через які bitonic sort є зручним для розпаралелювання. На основі проведеного аналізу було показано, що регулярність compare-swap операцій і фіксована послідовність шарів мережі створюють природну основу для побудови паралельного алгоритму.

У практичній частині роботи розроблено послідовну та паралельну реалізації алгоритму мовою Python. Послідовна версія була використана як базова для перевірки коректності та подальшого порівняння швидкодії. Паралельна реалізація побудована з використанням процесів і спільної пам'яті, що дало змогу виконувати незалежні compare-swap операції всередині окремих фаз алгоритму одночасно та зберегти логіку вихідного методу сортування. Проведене тестування підтвердило коректність обох реалізацій на граничних, типових і великих наборах даних.

Експериментальне дослідження показало, що паралельна реалізація не є доцільною для малих масивів через накладні витрати на створення процесів, синхронізацію та обмін даними. Водночас починаючи приблизно з `16 384` елементів вона починає демонструвати стійкий практичний виграш у швидкодії порівняно з послідовною версією. Найбільше прискорення в проведених експериментах було досягнуто для конфігурації з `8` процесами на масиві розміру `4 194 304` елементів і становило `4,642`, а найвища ефективність спостерігалася для `2` процесів і досягала `0,935`. Таким чином, встановлено, що мету роботи досягнуто: паралельну реалізацію алгоритму бітонічного сортування розроблено, протестовано й експериментально підтверджено її ефективність, причому отримане прискорення перевищує цільове значення `1,2` для достатньо великих обсягів даних.

Отримані результати можуть бути використані як основа для подальших досліджень паралельних методів сортування, зокрема для порівняння з іншими алгоритмами, для вивчення впливу архітектури обчислювальної системи на масштабованість, а також для подальшої оптимізації реалізації на багатоядерних CPU або інших платформах паралельних обчислень.
