# ЗАВДАННЯ

## Статус

Чернетка для перенесення в пояснювальну записку.

## Орієнтир по формату

- Розділ оформлюється як окрема сторінка з заголовком `ЗАВДАННЯ`.
- Основний зміст: 1 короткий вступний абзац + 8 нумерованих пунктів.
- Для цього розділу посилання на рисунки, таблиці й скріншоти не потрібні.

## Робочий текст розділу

Основним завданням курсової роботи є паралельна реалізація алгоритму бітонічного сортування мовою програмування Python, що забезпечує прискорення не менше 1,2 при достатньо великій складності обчислень.

1. Виконати огляд існуючих реалізацій алгоритму бітонічного сортування, послідовних та паралельних, з відповідними посиланнями на джерела інформації: статті, книги, електронні ресурси. Зробити висновок про актуальність дослідження.
2. Виконати розробку послідовного алгоритму бітонічного сортування у відповідності до обраної теми та мови програмування Python. Опис алгоритму забезпечити у вигляді псевдокоду. Провести тестування алгоритму та зробити висновок про коректність розробленого алгоритму. Дослідити швидкодію алгоритму при зростанні кількості даних та зробити висновки про доцільність паралельної реалізації алгоритму.
3. Виконати розробку паралельного алгоритму бітонічного сортування у відповідності до обраного завдання та мови програмування Python. Опис алгоритму забезпечити у вигляді псевдокоду. Забезпечити можливість запуску алгоритму на великих обсягах вхідних даних.
4. Виконати тестування алгоритму, що доводить коректність результатів обчислень. Тестування паралельного алгоритму обов'язково проводити на великій кількості даних. Коректність перевіряти порівнянням з результатами послідовного алгоритму.
5. Виконати дослідження швидкодії алгоритму при зростанні кількості даних для сортування.
6. Виконати експериментальне дослідження прискорення розробленого алгоритму при зростанні кількості даних для сортування. Реалізація алгоритму вважається успішною, якщо прискорення не менше 1,2.
7. Дослідити вплив параметрів паралельного алгоритму на отримуване прискорення. Одним з таких параметрів є кількість процесів, задіяних у паралельному виконанні, а також розмір задач, що розподіляються між ними.
8. Зробити висновки про переваги паралельної реалізації обчислень для алгоритму бітонічного сортування та програмних засобів, які використовувались у роботі.

## Нотатки для фінальної версії

- Якщо захочемо зробити формулювання точнішим, у першому абзаці можна замінити `мовою програмування Python` на `мовою програмування Python з використанням модуля multiprocessing та спільної пам'яті`.
- Пункт 7 вже адаптований під поточну реалізацію, де ключовими параметрами є кількість процесів і розбиття індексів на частини.
- За поточними бенчмарками в проєкті вимога щодо прискорення виконується на великих масивах, тому цей текст узгоджується з наявним кодом і результатами.
