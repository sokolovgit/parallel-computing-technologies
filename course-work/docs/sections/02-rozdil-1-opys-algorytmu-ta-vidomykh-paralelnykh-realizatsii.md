Бітонічне сортування є характерним прикладом алгоритму, в якому структура порівнянь наперед визначена і природно подається у вигляді сортувальної мережі. ### 1.1 Загальна характеристика алгоритму бітонічного сортування

Бітонічне сортування належить до алгоритмів сортування порівняннями та є класичним прикладом сортувальної мережі. Алгоритм був запропонований Кеннетом Бетчером [джерело: A02]. Його ідея полягає в тому, що довільну послідовність можна впорядковувати через побудову бітонічних послідовностей і їх подальше бітонічне злиття.

Бітонічною називають послідовність, яка спочатку монотонно зростає, а потім монотонно спадає, або може бути отримана циклічним зсувом із такої послідовності [джерела: A03, A04]. Для двох елементів така послідовність утворюється тривіально, а далі з малих бітонічних підпослідовностей послідовно формуються більші.

Особливість алгоритму полягає в тому, що послідовність компараторів задається наперед і залежить лише від розміру масиву. У термінах Кнута це відповідає oblivious sequence of comparisons, де структура порівнянь є фіксованою, а окремі компараторні модулі можна подати як вузли мережі [джерела: A02, A05].

У класичному варіанті алгоритм працює для кількості елементів, що дорівнює степеню двійки. Для масивів іншої довжини на практиці застосовують доповнення до найближчого значення вигляду `2^k`.

Класичне подання bitonic sort часто є рекурсивним: дві половини масиву впорядковуються у протилежних напрямках, утворюючи бітонічну послідовність, після чого виконується бітонічне злиття [джерела: A02, A03, A04]. Для практичної реалізації зручнішою є ітеративна форма, оскільки вона подає алгоритм як послідовність шарів compare-swap операцій. У такому записі використовуються параметри `k` і `j`, а партнер для індексу `i` визначається як `i XOR j`. Саме цей варіант краще підходить для програмної реалізації, побудови мережі та розподілу роботи між паралельними виконавцями.

Асимптотична складність бітонічного сортування в послідовному виконанні становить `O(n log^2 n)` [джерела: A03, A04]. Воно поступається оптимальним послідовним алгоритмам сортування, але компенсує це регулярною структурою, передбачуваним шаблоном доступу до даних і зручністю розпаралелювання. На кожному етапі значна частина compare-swap операцій є незалежною, тому вони можуть виконуватися одночасно, а синхронізація потрібна лише між окремими шарами мережі [джерела: A01, A03, A04].

### 1.2 Псевдокод алгоритму

Для опису алгоритму в цій роботі доцільно використати ітеративний псевдокод, який безпосередньо відповідає структурі сортувальної мережі та зручний для подальшого переходу до паралельної реалізації. Такий запис узгоджується з вимогами методичних рекомендацій: у першому рядку вказується назва алгоритму, далі подаються вхідні та вихідні дані, а самі дії описуються англійською мовою, близькою до природної.

`Рисунок 1.2 – Схема бітонічного злиття, у якій одна бітонічна послідовність розбивається на дві підпослідовності для подальшого впорядкування [джерело: A09].`

**Рекомендоване джерело рисунка 1.2:**  
слайд `Example: Bitonic Split` або `Bitonic Split` з `course-work/docs/materials/lec11-bitonicsort.pdf`

**Примітка:**  
цей рисунок краще використовувати як запозичену навчальну ілюстрацію, оскільки він пояснює саме ідею bitonic split, а не лише фінальну мережу компараторів.

```text
Algorithm Iterative bitonic sorting
Input: A - array of elements, n - number of elements in A, where n is a power of 2.
Output: array A sorted in ascending order.

1. For each block size k = 2, 4, ..., n:
2.     For each stride j = k / 2, k / 4, ..., 1:
3.         For each index i = 0, ..., n - 1:
4.             Compute partner <- i XOR j.
5.             If partner <= i, continue to the next index.
6.             If (i AND k) = 0 and A[i] > A[partner]:
7.                 swap A[i] and A[partner].
8.             If (i AND k) != 0 and A[i] < A[partner]:
9.                 swap A[i] and A[partner].
10. Return A.
```

У цьому псевдокоді цикл за `k` визначає розмір блока, а цикл за `j` - окремі етапи порівняння в межах цього блока. Для кожного індексу `i` обчислюється парний індекс `partner`, після чого виконується compare-swap у потрібному напрямку. Такий запис зручний тим, що майже безпосередньо переходить у програмну реалізацію і наочно показує незалежність порівнянь усередині одного шару мережі.

### 1.3 Відомі паралельні реалізації алгоритму

Історично бітонічне сортування розглядалося як алгоритм, зручний для паралельного виконання на обчислювальних структурах із регулярною топологією [джерела: A01, A02, A05]. Регулярність структури компараторів дає змогу реалізовувати його як на SIMD-подібних моделях, так і на багатопроцесорних або багатопотокових системах.

Найбільш загальна ідея паралельної реалізації полягає в тому, що всі compare-swap операції одного шару мережі виконуються одночасно. Після завершення шару виконується синхронізація, і лише тоді починається наступний етап [джерела: A03, A04].

У літературі можна виділити кілька типових підходів до паралельної реалізації bitonic sort. Перший полягає у прямому відображенні компараторів мережі на паралельні обчислювальні елементи, де кожен потік або процес виконує свою підмножину compare-swap операцій [джерела: A02, A03, A05]. Другий підхід характерний для систем зі спільною пам’яттю, де виконавцям передаються діапазони індексів, а синхронізація здійснюється між окремими шарами мережі. Третій підхід орієнтований на GPU та архітектури масового паралелізму, де один шар мережі виконується великою кількістю потоків одночасно [джерела: A03, A09].

Окремий інтерес становлять CUDA-реалізації алгоритму. У відповідній публікації розглядається підхід, де продуктивність підвищується за рахунок використання shared memory та зменшення кількості kernel launches [джерело: A11]. Для цієї роботи це джерело корисне як приклад практичного напряму оптимізації паралельних реалізацій, хоча його результати не варто безпосередньо порівнювати з реалізацією на Python.

З погляду дослідження паралельних обчислень бітонічне сортування є зручним тому, що має регулярну структуру, легко візуалізується у вигляді мережі та допускає природний розподіл compare-swap операцій між паралельними виконавцями.

Отже, огляд відомих підходів показує, що бітонічне сортування є типовим прикладом алгоритму, в якому паралелізм закладений у саму структуру обчислень. Саме тому воно є доцільним об’єктом для побудови та експериментального дослідження власної паралельної реалізації мовою Python.

### 1.4 Висновки до розділу

Бітонічне сортування є класичним прикладом сортувальної мережі, в якій структура порівнянь задається наперед і не залежить від конкретних значень елементів. Його регулярна будова робить алгоритм зручним для подання у вигляді послідовних шарів compare-swap операцій і придатним до паралельної реалізації. Саме тому далі доцільно перейти до побудови власної послідовної реалізації як базового варіанту для подальшого аналізу швидкодії.
