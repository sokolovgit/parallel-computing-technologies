# ВСТУП

## Статус

Чернетка для перенесення в пояснювальну записку.

## Орієнтир по формату

- Розділ має займати не менше 1 сторінки.
- У вступі потрібно:
  - показати актуальність паралельних обчислень;
  - звузити актуальність до алгоритму бітонічного сортування;
  - коротко зазначити, що саме буде реалізовано і досліджено в роботі.
- Для цього розділу ілюстрації не є обов'язковими.

## Робочий текст розділу

Головним завданням курсової роботи є паралельна реалізація алгоритму бітонічного сортування мовою програмування Python, яка забезпечує прискорення не менше 1,2 при достатньо великій складності обчислень.

Сучасні обчислювальні системи дедалі ширше використовують багатоядерні та багатопроцесорні архітектури, тому розробка ефективних паралельних алгоритмів є важливою складовою високопродуктивних обчислень. Паралельне виконання дає змогу скоротити час обробки великих обсягів даних за рахунок одночасного виконання незалежних або частково незалежних підзадач, однак ефективність такого підходу безпосередньо залежить від структури алгоритму, способу розподілу роботи та накладних витрат на синхронізацію [джерело: A01].

Однією з базових задач, для яких активно досліджуються паралельні методи, є сортування даних. Сортування використовується як самостійна операція, а також як складова більш складних алгоритмів обробки, аналізу та впорядкування інформації. Для паралельної реалізації особливий інтерес становлять алгоритми, структура яких заздалегідь визначає послідовність порівнянь і перестановок елементів. До таких алгоритмів належить бітонічне сортування, запропоноване К. Бетчером у межах підходу сортувальних мереж [джерело: A02].

Алгоритм бітонічного сортування базується на побудові бітонічних послідовностей та їх подальшому злитті за фіксованою схемою компараторів. Завдяки регулярній структурі та чітко визначеним етапам порівняння цей алгоритм є зручним для паралельного виконання, оскільки значна частина операцій одного етапу може виконуватися одночасно [джерела: A02, A03, A04]. Саме тому бітонічне сортування є зручним прикладом для дослідження особливостей побудови паралельного алгоритму, організації синхронізації та оцінювання досягнутого прискорення.

У межах цієї курсової роботи буде розроблено послідовну та паралельну реалізації алгоритму бітонічного сортування мовою Python. Послідовна версія алгоритму використовуватиметься як базовий варіант для перевірки коректності результатів і для подальшого порівняння швидкодії. Паралельна версія буде побудована на основі тих самих операцій порівняння та обміну елементів, але з розподілом індексів між кількома процесами та синхронізацією виконання на окремих етапах сортування.

У роботі буде проведено тестування обох реалізацій, досліджено їхню швидкодію на масивах різного розміру, а також проаналізовано вплив кількості процесів на отримане прискорення. Очікуваним результатом є підтвердження доцільності паралельної реалізації алгоритму бітонічного сортування для великих обсягів даних і визначення параметрів запуску, за яких досягається найбільша ефективність.

## Нотатки для фінальної версії

- Посилання виду `A01`, `A02` тощо треба буде замінити на звичайні бібліографічні посилання після фінального узгодження списку джерел.
- Формулювання вступу вже узгоджене з поточним кодом: у проєкті реалізовано окремі послідовну й паралельну версії алгоритму, а також бенчмарки для різної кількості процесів.
- Якщо додамо офіційну документацію Python для `multiprocessing` та `shared_memory`, розділи про реалізацію можна буде підсилити точнішими технічними посиланнями.
