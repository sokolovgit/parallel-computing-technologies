# ВСТУП

Головним завданням курсової роботи є паралельна реалізація алгоритму бітонічного сортування мовою програмування Python, яка забезпечує прискорення не менше 1,2 при достатньо великій складності обчислень.

Сучасні обчислювальні системи дедалі ширше використовують багатоядерні та багатопроцесорні архітектури, тому розробка ефективних паралельних алгоритмів є важливою складовою високопродуктивних обчислень. Паралельне виконання дає змогу скоротити час обробки великих обсягів даних за рахунок одночасного виконання незалежних або частково незалежних підзадач, однак ефективність такого підходу безпосередньо залежить від структури алгоритму, способу розподілу роботи та накладних витрат на синхронізацію [1].

Однією з базових задач, для яких активно досліджуються паралельні методи, є сортування даних. Сортування використовується як самостійна операція, а також як складова більш складних алгоритмів обробки, аналізу та впорядкування інформації. Для паралельної реалізації особливий інтерес становлять алгоритми, структура яких заздалегідь визначає послідовність порівнянь і перестановок елементів. До таких алгоритмів належить бітонічне сортування, запропоноване К. Бетчером у межах підходу сортувальних мереж [2].

Алгоритм бітонічного сортування базується на побудові бітонічних послідовностей та їх подальшому злитті за фіксованою схемою компараторів. Завдяки регулярній структурі та чітко визначеним етапам порівняння цей алгоритм є зручним для паралельного виконання, оскільки значна частина операцій одного етапу може виконуватися одночасно [2, 3, 4]. Саме тому бітонічне сортування є зручним прикладом для дослідження особливостей побудови паралельного алгоритму, організації синхронізації та оцінювання досягнутого прискорення.

У межах цієї курсової роботи буде розроблено послідовну та паралельну реалізації алгоритму бітонічного сортування мовою Python. Послідовна версія алгоритму використовуватиметься як базовий варіант для перевірки коректності результатів і для подальшого порівняння швидкодії. Паралельна версія буде побудована на основі тих самих операцій порівняння та обміну елементів, але з розподілом індексів між кількома процесами та синхронізацією виконання на окремих етапах сортування.

У роботі буде проведено тестування обох реалізацій, досліджено їхню швидкодію на масивах різного розміру, а також проаналізовано вплив кількості процесів на отримане прискорення. Очікуваним результатом є підтвердження доцільності паралельної реалізації алгоритму бітонічного сортування для великих обсягів даних і визначення параметрів запуску, за яких досягається найбільша ефективність.
